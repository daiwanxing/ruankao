# 第三章 信息网络系统

## 3.1 信息网络系统体系框架和OSI七层模型

### 一、教学目标

掌握信息网络系统体系框架的五大部分和OSI七层模型各层的功能及特点。

### 二、核心讲解

#### 1. 信息网络系统体系框架模型

| 组成部分 | 内容 |
|----------|------|
| 网络传输平台 | 包括传输、路由、交换、有线和无线接入等设备和系统 |
| 网络和应用服务平台 | 包括域名解析系统(DNS)、地址分配系统、业务应用系统(OA、WWW、电子邮件、语音会议、视频会议、VOD、人脸识别等) |
| 安全服务平台 | 包括信息加解密、防火墙、入侵检测、漏洞扫描、病毒查杀、安全审计、数字证书等 |
| 网络管理和维护平台 | 负责整个信息网络系统的管理和维护，如果对外提供业务服务，还需要专门的运营系统 |
| 环境系统 | 包括机房建设、环境监控、智能安防、节能降耗、综合布线等 |

#### 2. OSI七层模型

| 层级 | 含义 | 主要功能 |
|------|------|----------|
| **物理层** | 最底层，规定承载各层发送和接收具体数据的物理硬件方法 | 定义物理设备之间的基本数据传输方法（路由器、交换机、传输设备等） |
| **数据链路层** | 将物理层透明传输过来的比特流组织成有意义的数据包 | 规定数据包格式和大小、寻址方式、同步控制、差错控制、流量控制 |
| **网络层** | 定义和规范不同网络间的通信规则 | 寻址和路由选择、链路连接的建立、保持和终止 |
| **传输层** | 为会话层提供建立可靠的端到端的透明数据传输机制 | 定义跨网络的逻辑连接、完成差错纠正和流量控制 |
| **会话层** | 向两个表示层实体提供建立、管理、拆除和使用连接的方法 | 建立、管理、拆除会话连接 |
| **表示层** | 数据翻译、格式化、语法选择 | 数据编解码、加密解密、数据格式转换、数据压缩、语法翻译 |
| **应用层** | 最顶层，直接向用户提供信息通信服务 | 直接向用户提供服务（HTTP、FTP、SMTP、WWW、邮件、视频会议、游戏等） |

**记忆口诀**："**物**理**数**据**网**，**传**输**会**表**应**"
- 物（物理层）
- 理（数据链路层）
- 数（网络层）
- 据（传输层）
- 网（会话层）
- 传（表示层）
- 输（应用层）

### 三、通俗例子

**OSI七层模型例子**：
就像寄快递：
- **物理层**：快递员、交通工具、道路
- **数据链路层**：快递包装、收件人地址格式
- **网络层**：选择快递路线（顺丰、圆通等）
- **传输层**：确保快递安全送达
- **会话层**：确认收货、签收过程
- **表示层**：快递单语言翻译、货物包装方式
- **应用层**：快递的具体服务内容（文件、物品等）

**信息网络系统框架例子**：
就像一个城市的基础设施：
- **网络传输平台**：道路、交通工具
- **网络和应用服务平台**：邮局、商店、银行等服务场所
- **安全服务平台**：警察局、保安公司、监控系统
- **网络管理和维护平台**：交通管理中心、市政管理部门
- **环境系统**：城市规划、建筑设计、环境监测

### 四、知识点速记

1. **信息网络系统五平台**：网络传输、网络应用服务、安全服务、网络管理维护、环境系统
2. **OSI七层模型**：物理层→数据链路层→网络层→传输层→会话层→表示层→应用层
3. **物理层**：硬件基础，定义物理传输方法
4. **数据链路层**：数据包格式，寻址和差错控制
5. **网络层**：路由选择，跨网络通信
6. **传输层**：端到端连接，可靠传输
7. **会话层**：建立和管理会话
8. **表示层**：数据转换和翻译
9. **应用层**：用户服务，直接交互

## 3.2 TCP/IP协议族

### 一、教学目标

掌握TCP/IP协议族的四层结构和主要协议，理解IPv4和IPv6的区别。

### 二、核心讲解

#### 1. TCP/IP协议族

| 层级 | 主要协议 | 功能说明 |
|------|----------|----------|
| **应用层** | Telnet、FTP、SMTP等 | 接收来自传输层的数据，或按不同的应用要求将数据传输至传输层 |
| **传输层** | UDP、TCP | TCP是面向连接的协议，在收发数据前必须和对方建立可靠的连接；UDP是非连接协议，传输数据前源端和终端不建立连接 |
| **互联网络层** | ICMP、IP、IGMP | 负责网络中数据包的具体传输，最基本的是IPv4和IPv6 |
| **物理和数据链路层** | ARP、RARP | 提供链路管理错误检测、对不同通信媒介的信息细节进行处理 |

#### 2. IPv4协议

**地址结构**：
- 由32位二进制数组成，即4个字节
- 点分十进制表示法（如192.121.123.56）
- 由网络位和主机位两大部分组成

**地址分类**：
- **A类地址**：适用于大型网络建设，支持126个网络，每个网络最多支持16,777,214个主机地址
- **B类地址**：适用于中型网络建设，支持16,384个网络，每个网络最多支持65,534个主机地址
- **C类地址**：适用于小型网络建设，支持209万余个网络，每个网络最多支持254个主机地址
- **D类地址**：专门用于组播地址
- **E类地址**：保留地址

**NAT技术**：
- 一般在家庭网关、企业网关或者接口路由器等设备上实现
- 使用私网地址的主机需要通过地址转换技术（NAT）与公网IPv4地址的主机进行通信

**路由类型**：
- **直连路由**：直接连接的网段
- **静态路由**：人工配置的路由
- **动态路由**：路由器自动学习生成的路由

#### 3. IPv6协议

**特点**：
- 由128位二进制数组成，是IPv4地址长度的4倍
- 解决IPv4地址不够问题

### 三、通俗例子

**TCP和UDP的例子**：
- **TCP**：像打电话，需要先拨号建立连接，确认对方接听才能通话，通话过程中会确认对方是否听到（可靠传输）
- **UDP**：像广播喇叭，直接喊话，不管有没有人听到，也不需要确认（不可靠传输，但速度快）

**IP地址分类例子**：
- **A类**：像大型集团公司，总部（网络号）+各地分公司（主机号），分公司数量少但每个分公司人数很多
- **B类**：像中型公司，分公司数量中等，每个分公司人数中等
- **C类**：像小型商店，分公司数量很多，但每个分公司人数很少

### 四、知识点速记

1. **TCP/IP四层**：应用层→传输层→互联网络层→物理和数据链路层
2. **TCP特点**：面向连接、可靠传输
3. **UDP特点**：无连接、不可靠传输、速度快
4. **IPv4地址**：32位，分A、B、C、D、E五类
5. **IPv6地址**：128位，解决IPv4地址不足问题
6. **路由三类型**：直连、静态、动态

## 3.3 网络传输平台

### 一、教学目标

掌握网络传输平台的一般架构和主要技术、运营商网络架构、4G/5G移动通信技术以及物联网组网技术。

### 二、核心讲解

#### 1. 网络传输平台的一般架构和主要技术

**定义**：负责信息的传输，一般由传输媒介、传输设备、路由设备、交换设备、有线接入设备、无线接入设备和相关系统组成

**网络传输媒介**：
- **有线媒介**：光纤、双绞线、同轴电缆等
- **无线媒介**：长波（3～30kHz）、中波（0.03～3MHz）、短波（3~30MHz）、超短波（30～300MHz）、微波（0.3~300GHz）等

**网络传输技术**：
- **基于光纤**：SDH（同步数字序列）、PDH（准同步数字序列）、DWDM（密集波分复用）
- **基于同轴电缆**：HFC（混合光纤同轴电缆）
- **基于无线媒介**：Wi-Fi、数字微波通信DMC、卫星小数据站VSAT、数字卫星通信系统、2G到6G移动通信系统

**按物理覆盖和管理范畴划分组网**：

| 分类 | 内容 |
|------|------|
| **局域网（LAN）** | 早期的局域网技术包括以太网、令牌环网、光纤分布式数据网等，目前基本采用各类以太网交换机组建局域网，包括百兆、千兆、万兆以太网交换机。无线局域网Wi-Fi作为局域网的无线接入 |
| **城域网（MAN）** | 一个城市范围内建设的网络 |
| **广域网（WAN）** | 跨地区、跨省市、跨国家的更大规模网络的统称，用来连接地区的城域网、省网和各个国家的网络，Internet是全球最大的广域网 |

**VLAN**：一种将局域网设备从逻辑上划分成一个个虚拟网段（更小的局域网）

**有线、无线接入技术**：
- **有线接入**：光纤接入网（OAN），无源光网络（PON）逐步获得广泛应用，包括EPON（以太网无源光网络）、GPON（千兆无源光网络）、10G-PON（10G无源光网络）
- **无线接入**：Wi-Fi、蓝牙等

#### 2. 运营商网络架构

**组成**：全国骨干网、省级骨干网、城域网、接入网

**城域网三层架构**：
- **核心层**：路由器设备
- **汇聚层**：汇聚交换机设备
- **接入层**：面向各类园区、楼宇、住宅小区等商业、家庭和个人用户

#### 3. 4G/5G移动通信

**第1代**：模拟蜂窝通信系统
**第2代**：数字蜂窝移动通信系统
**第3代**：数字移动通信系统
**第4代**：高速率、高容量；网络频谱更宽；智能性能更高；兼容性能更平滑；更高质量、更低费用的通信；更好安全性

**5G移动通信**：

**(1) 三大类应用场景**：
- 增强移动宽带
- 超高可靠低时延通信
- 海量机器类通信

**(2) 主要性能指标**：
- 峰值速率达到10～20Gbit/s，满足高清视频、虚拟现实等大数据量传输
- 空中接口时延低至1ms，满足自动驾驶、远程医疗等实时应用
- 具备百万连接/平方公里的设备连接能力，满足物联网通信
- 频谱效率比4G LTE提升3倍以上
- 连续广域覆盖和高移动性下，用户体验速率达到100Mbit/s
- 流量密度达到10Mbps/m²以上
- 移动性支持500km/h的高速移动

#### 4. 物联网组网技术

**架构三层**：

| 层级 | 关键技术 | 作用 |
|------|----------|------|
| **感知层** | 传感器技术、射频识别技术、条码、二维码技术、GPS技术、NFC技术、微机电系统技术等 | 采集和感知物理世界的信息 |
| **网络层** | - | 负责连通感知层和应用层，安全、顺畅地传输数据和指令 |
| **应用层** | - | 承载着物联网应用的信息数据和业务处置逻辑，对感知层收集到的信息数据进行处理，对感知层下达相关处置指令 |

### 三、通俗例子

**LAN、MAN、WAN例子**：
- **LAN**：像家里或办公室的Wi-Fi网络，覆盖范围小
- **MAN**：像整个城市的公交网络，覆盖一个城市
- **WAN**：像全国的高铁网络，连接不同城市

**5G应用场景例子**：
- **增强移动宽带**：VR游戏、8K视频观看
- **超高可靠低时延通信**：自动驾驶汽车、远程手术
- **海量机器类通信**：智能家居、智能农业、智慧城市

**物联网三层例子**：
智能门锁：
- **感知层**：指纹识别、密码输入、门锁状态检测
- **网络层**：Wi-Fi、蓝牙连接手机APP
- **应用层**：手机APP显示门锁状态、远程开锁、记录开锁历史

### 四、知识点速记

1. **网络三分类**：LAN（局域网）、MAN（城域网）、WAN（广域网）
2. **PON三类型**：EPON、GPON、10G-PON
3. **城域网三层**：核心层（路由器）、汇聚层（汇聚交换机）、接入层（面向用户）
4. **移动通信五代**：1G（模拟）、2G（数字）、3G（移动数字）、4G（高速移动）、5G（超高速）
5. **5G三场景**：增强移动宽带、超高可靠低时延、海量机器类通信
6. **物联网三层次**：感知层（传感器）、网络层（传输）、应用层（处理）

## 3.5 安全服务平台

### 一、教学目标

掌握信息网络系统安全体系框架的六个方面和云计算安全。

### 二、核心讲解

#### 信息网络系统安全体系框架

| 安全方面 | 解释 |
|----------|------|
| **物理安全** | 包括环境安全、供配电系统、安防系统和消防系统四个部分 |
| **网络安全** | 网络结构；访问控制；网络入侵防护；网络安全审计。网络安全设备和手段主要包括：防火墙、利用Vlan等技术进行安全域划分、入侵检测系统/入侵防御系统、网络安全审计 |
| **主机安全** | 身份鉴别；访问控制；安全漏洞和攻击防御；主机安全审计等 |
| **应用安全** | 身份和访问控制；应用安全漏洞；会话管理等 |
| **数据安全** | 机密数据保护；数据备份恢复等 |
| **安全管理** | 安全组织和责任；风险管理工作机制；应急处理工作机制；容灾备份工作机制；制定系统上线、切换办法、安全运维方案等 |

#### 云计算安全

**主要内容**：业务平台迁移到云计算环境后的主机、网络、数据安全、DDoS攻击

### 三、通俗例子

**安全体系例子**：
就像保护一栋大楼：
- **物理安全**：大楼建筑、供电、安保、消防
- **网络安全**：大楼的门禁系统、网络隔离、入侵检测
- **主机安全**：房间内的安全门锁
- **应用安全**：房间内贵重物品的保险箱
- **数据安全**：重要文件的加密和备份
- **安全管理**：大楼的管理制度、应急预案

### 四、知识点速记

1. **安全六方面**：物理、网络、主机、应用、数据、管理
2. **物理安全四要素**：环境、供配电、安防、消防
3. **网络安全手段**：防火墙、Vlan、IDS/IPS、审计
4. **主机安全四要素**：鉴别、访问控制、漏洞防御、审计
5. **应用安全三要素**：身份访问控制、漏洞、会话管理
6. **数据安全两要素**：机密保护、备份恢复

## 3.6 网络管理和维护平台

### 一、教学目标

掌握网络管理的五大功能和管理系统的功能体系结构。

### 二、核心讲解

#### 网络管理的五大功能

1. **故障管理**：检测、隔离、纠正网络故障
2. **配置管理**：网络设备的配置信息管理
3. **性能管理**：监测网络性能，优化网络性能
4. **计费管理**：网络使用情况的计费和统计
5. **安全管理**：网络访问控制和安全审计

**记忆口诀**："**故**障**配**置**性**计**安**"

#### 网络管理和维护系统

**功能体系结构**：
- **网元/网络层**（最底层）：具体的网络设备和网络
- **管理应用层**：网络管理系统
- **表示层**：用户界面

### 三、通俗例子

**网络管理五大功能例子**：
就像物业管理：
- **故障管理**：发现电梯坏了（故障）→联系维修→修好
- **配置管理**：记录每户的水电表配置信息
- **性能管理**：定期检查电梯运行速度、楼道照明亮度
- **计费管理**：计算每户的水电费、物业费
- **安全管理**：门禁系统、监控录像、访客登记

### 四、知识点速记

1. **网络管理五功能**：故障、配置、性能、计费、安全
