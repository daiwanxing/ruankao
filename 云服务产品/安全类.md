# 安全类产品（软考监理可背版）

目标：看到题干场景，能在 3 秒内选对产品。

## 1. 一页速记（先背这个）

### 1.1 两大高频：DDoS 高防 IP vs WAF

| 产品 | 防什么（本质） | 题干“秒杀关键词” | 最典型场景 |
| --- | --- | --- | --- |
| **DDoS 高防 IP** | **抗大流量攻击**（把“脏流量”清洗后再回源） | **带宽被打满**、**访问被打挂**、**连接数暴涨**、**流量清洗**、**抗攻击**、**实时性要求高** | 金融、游戏、媒资、电商、政府等“不能停”的在线业务 |
| **WAF（Web 应用防火墙）** | **护 Web 业务**（HTTP/HTTPS 应用层攻击防护） | **SQL 注入**、**XSS**、**Web 攻击**、**恶意爬虫**、**Web 安全防护**、**业务安全要求高** | 金融、电商、O2O、互联网+、游戏、政府、保险等网站/接口防护 |

**一句话口诀：**
- **DDoS = 洪水冲垮道路（流量把链路/带宽打爆）**
- **WAF = 撬门入室（对网站/接口下手：注入、脚本、爬虫）**

### 1.2 做题套路（监理常见问法）
- 题目先给“行业 + 现象”，让你选产品：如“金融业务实时性高 + 访问被打挂”→ 优先 **DDoS 高防 IP**
- 题目给“攻击类型”，让你选防护：如“SQL 注入 / XSS”→ 秒选 **WAF**
- 题目给“云上云外客户都能用”→ 说明它是对公网服务能力（常见于 **DDoS 高防 IP**、**WAF** 的对外服务描述）

## 2. DDoS 高防 IP（背诵卡片）

### 2.1 定义（背一行）
**DDoS 高防 IP**：面向公网业务的抗 DDoS 服务，通过**牵引与清洗**抵御大流量攻击，保证业务可用。

### 2.2 你必须会的 4 个点
1. **解决的是“可用性”问题**：被打就“打不开”，优先想到它。
2. **典型现象**：带宽被占满、连接耗尽、请求洪峰导致服务不可达。
3. **典型行业/业务**：金融、娱乐（游戏）、媒资、电商、政府等**实时性要求高**的业务。
4. **监理审查点（口头会问）**：是否有清洗能力与回源方案、是否支持弹性防护、是否有告警与应急预案。

### 2.3 易错点
- **DDoS 高防 IP 不等于 WAF**：它主要抗“流量型/资源耗尽型”攻击；题干只要把“带宽、洪峰、打挂”说得很明确，优先选它。

---

## 3. WAF（Web 应用防火墙）（背诵卡片）

### 3.1 定义（背一行）
**WAF**：部署在 Web 业务入口，针对 HTTP/HTTPS 的**应用层攻击**进行检测与拦截，保护网站与接口安全。

### 3.2 你必须会的 4 个点
1. **防的是“Web 攻击”**：SQL 注入、XSS、恶意爬虫、越权访问等题干常见。
2. **题干出现“网站/接口安全要求高”**：金融、电商、O2O、政府、保险等。
3. **监理审查点**：是否覆盖关键域名/API，策略是否能按业务定制，是否有日志审计与告警。
4. **与 DDoS 的分界线**：题干如果明确描述“攻击类型是注入/脚本”，优先 WAF；如果描述“被打到打不开/带宽打满”，优先 DDoS 高防 IP。

---

## 4. 选择题秒杀模板（把题干翻译成关键词）

### 4.1 看到这些词，直接选 DDoS 高防 IP
- “带宽打满、线路被打爆、访问被打挂、服务不可达、流量清洗、抗 DDoS”

### 4.2 看到这些词，直接选 WAF
- “SQL 注入、XSS、Web 攻击、网站安全防护、恶意爬虫、HTTP/HTTPS 防护”
