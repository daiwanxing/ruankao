# 第一篇导学：把“知识点”串成“工程监理脑图”

## 1. 这篇讲什么（对应考纲与监理视角）

第一篇的主题是“信息系统工程知识”，核心任务是把信息系统从“宏观环境 → 工程建设 → 网络/资源/应用 → 安全 → 运维”的全链路讲清楚。

对监理师而言，这些知识不是为了会写代码，而是为了：

- **看得懂方案**：能判断架构/网络/数据/安全/运维方案是否合理。
- **抓得住风险**：知道哪些环节最容易“验收不过/出事故/超预算/延期”。
- **问得出关键问题**：能对承建方提出可验证的质量、进度、成本与安全要求。

## 2. 全篇主线（建议你背下来）

1) **信息化发展**：为什么要做信息化？底座有哪些？（云、大数据、物联网、区块链、AI 等）  
2) **信息系统工程**：怎么把系统“按工程”建出来？（系统工程方法 + 软件工程 + 数据工程 + 集成 + 安全工程）  
3) **信息网络系统**：网络怎么搭？协议怎么跑？平台怎么分？  
4) **信息资源系统**：算力/存储/数据库/云资源怎么选？  
5) **信息应用系统**：企业/政府常见系统类型与典型系统（TPS/MIS/DSS/OA/ES…）  
6) **信息安全**：属性、技术措施、等保、数据安全  
7) **运行维护**：ITSS/ITSM、服务交付、SLA、应急

你会发现：**第2章是“工程方法论中枢”**，它把后面网络/资源/应用/安全/运维都串起来。

## 3. 学习方法（按题型反推）

- **选择题**：重概念边界与对比（如 IaaS/PaaS/SaaS、SAN/NAS、白盒/黑盒、蓝绿/金丝雀、等保级别）。
- **监理场景题/综合题（如果遇到）**：重“监理怎么做”——看文档、查过程、核指标、留证据、能验收。

建议你把每个小结都按“三问”训练：

1) 这是什么（定义/范围）？
2) 解决什么问题（价值/场景）？
3) 怎么验证它做到了（指标/证据/流程）？

## 4. 课后作业（导学）

1) 用一张纸画出本篇主线（7章）并写出每章的关键词（每章 5 个）。  
2) 把“监理三问”抄到笔记首页：**定义-价值-验证**。  
3) 预习：阅读 `讲义讲解/第一篇-备课/01-信息化发展.md` 的“信息与信息化”小结，尝试用自己的话解释“DIKW”。
