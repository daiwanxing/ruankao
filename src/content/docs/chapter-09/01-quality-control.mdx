## 9.1 质量管理基本概念

### 一、教学目标

掌握质量管理的基本概念、质量控制方法和质量保证体系。

### 二、核心讲解

#### 1. 质量的定义

质量是一组固有特性满足要求的程度。

#### 2. 质量管理三大过程

<MemoryCard 
  title="质量管理三过程"
  formula="质量规划→质量保证→质量控制"
  importance={5}
  category="formula"
/>

| 过程 | 说明 | 主要活动 |
|------|------|----------|
| **质量规划** | 确定质量标准和要求 | 制定质量计划、质量标准 |
| **质量保证** | 确保按标准执行 | 质量审计、过程分析 |
| **质量控制** | 监控质量结果 | 检查、测试、评审 |

<Card title="PDCA四阶段" icon="information">
- **Plan（计划）**：制定目标和计划
- **Do（执行）**：实施计划
- **Check（检查）**：检查执行效果

## 9.4 软件质量模型

### 一、核心讲解

#### 1. ISO/IEC 25010质量模型

| 质量特性 | 子特性 |
|----------|--------|
| **功能性** | 功能完备性、功能正确性、功能适宜性 |
| **性能效率** | 时间特性、资源利用性、容量 |
| **兼容性** | 共存性、互操作性 |
| **易用性** | 可辨识性、易学性、易操作性 |
| **可靠性** | 成熟性、可用性、容错性、易恢复性 |
| **安全性** | 保密性、完整性、抗抵赖性 |
| **可维护性** | 模块化、可重用性、易分析性 |
| **可移植性** | 适应性、易安装性、易替换性 |

## 9.5 验收管理

### 一、核心讲解

#### 1. 验收类型

<MemoryCard 
  title="验收四阶段"
  formula="单元测试→集成测试→系统测试→验收测试"
  importance={5}
  category="formula"
/>

| 测试阶段 | 测试对象 | 测试者 |
|----------|----------|--------|
| **单元测试** | 模块或函数 | 开发人员 |
| **集成测试** | 模块组合 | 开发/测试人员 |
| **系统测试** | 完整系统 | 测试团队 |
| **验收测试** | 完整系统 | 用户/第三方 |

#### 2. 验收测试类型

| 类型 | 说明 |
|------|------|
| **α测试** | 开发环境下由用户测试 |
| **β测试** | 实际环境下由最终用户测试 |

#### 3. 验收流程

<Card title="验收六步骤" icon="information">
1. **验收准备**：制定验收计划、准备验收资料
2. **验收测试**：执行测试用例
3. **问题处理**：记录并跟踪问题
4. **试运行**：实际环境运行一段时间
5. **验收评审**：评审验收结果
6. **验收报告**：编制验收报告、签署验收证书
</Card>

## 重点总结

### 本章必记考点 ⭐⭐⭐⭐⭐

1. **质量三过程**：质量规划→质量保证→质量控制
2. **QC七工具**：因果图、直方图、控制图、检查表、散点图、排列图、分层法
3. **PDCA循环**：Plan→Do→Check→Act
4. **ISO 9000八原则**：以客户为中心、领导作用、全员参与等
5. **测试四阶段**：单元→集成→系统→验收
6. **α/β测试**：α(开发环境/用户)、β(实际环境/最终用户)
7. **验收流程**：准备→测试→问题处理→试运行→评审→报告
