## 12.1 项目文档管理

### 一、教学目标

掌握信息系统工程项目文档体系、编制要求和管理规范。

### 二、核心讲解

#### 1. 文档分类

<MemoryCard 
  title="文档三大类"
  formula="开发文档、产品文档、管理文档"
  importance={5}
  category="formula"
/>

| 文档类型 | 说明 | 主要文档 |
|----------|------|----------|
| **开发文档** | 描述开发过程 | 可行性研究报告、需求规格说明书、设计说明书、测试报告 |
| **产品文档** | 描述产品使用 | 用户手册、操作手册、维护手册 |
| **管理文档** | 项目管理相关 | 项目计划、进度报告、质量报告、变更申请 |

#### 2. 主要文档及编制阶段

<Card title="项目各阶段文档" icon="information">
**立项阶段**：
- 项目建议书
- 可行性研究报告
- 项目任务书

**需求分析阶段**：
- 需求规格说明书
- 数据要求说明书

**设计阶段**：
- 概要设计说明书
- 详细设计说明书
- 数据库设计说明书

**实施阶段**：
- 编码规范
- 测试计划
- 测试报告

**验收阶段**：
- 用户手册
- 操作手册
- 项目总结报告
</Card>

## 12.2 配置管理

### 一、核心讲解

#### 1. 配置管理基本概念

**配置项(CI)**：受配置管理控制的项目元素，如代码、文档、数据等。

**基线(Baseline)**：经过评审和批准的配置项集合，作为进一步开发的基础。

#### 2. 配置管理活动

<MemoryCard 
  title="配置管理五活动"
  formula="制定CM计划、配置标识、配置控制、配置状态记录、配置审核"
  importance={5}
  category="formula"
/>

| 活动 | 说明 |
|------|------|
| **制定CM计划** | 制定配置管理计划 |
| **配置标识** | 识别和命名配置项 |
| **配置控制** | 控制配置项的变更 |
| **配置状态记录** | 记录和报告配置项状态 |
| **配置审核** | 验证配置项的一致性和完整性 |

#### 3. 版本控制

<Card title="版本号命名规范" icon="information">
**X.Y.Z格式**：
- **X（主版本号）**：重大功能变更
- **Y（次版本号）**：新增功能
- **Z（修订号）**：bug修复

示例：v2.3.5 → v2.3.6（bug修复）→ v2.4.0（新功能）→ v3.0.0（重大变更）
</Card>

## 12.3 监理文档

### 一、核心讲解

#### 1. 监理文档体系

<MemoryCard 
  title="监理三类文档"
  formula="监理规划类、监理实施类、监理总结类"
  importance={5}
  category="key-point"
/>

| 文档类型 | 主要文档 | 编制时间 |
|----------|----------|----------|
| **监理规划类** | 监理规划、监理实施细则 | 项目启动前 |
| **监理实施类** | 监理日志、监理月报、监理通知、会议纪要 | 项目实施期间 |
| **监理总结类** | 阶段性报告、监理工作总结 | 阶段结束或项目结束 |

#### 2. 主要监理文档

**监理规划**：
- 项目概况
- 监理工作依据
- 监理工作内容
- 监理工作方法及措施
- 监理工作制度
- 监理组织机构

**监理实施细则**：
- 专业工程特点
- 监理工作流程
- 监理工作要点
- 监理工作方法

## 12.4 资料归档

### 一、核心讲解

#### 1. 归档文件分类

<Card title="归档文件九类" icon="information">
1. **立项文件**：项目建议书、可行性研究报告
2. **合同文件**：合同、协议
3. **监理文件**：监理规划、监理日志、监理报告
4. **设计文件**：需求规格说明、设计说明书
5. **测试文件**：测试计划、测试报告
6. **验收文件**：验收申请、验收报告
7. **培训文件**：培训计划、培训记录
8. **变更文件**：变更申请、变更审批
9. **其他文件**：会议纪要、往来函件
</Card>

#### 2. 归档要求

<MemoryCard 
  title="归档原则"
  formula="真实、准确、完整、系统"
  importance={4}
  category="key-point"
/>

**归档要求**：
- 文件齐全完整
- 签字盖章齐全
- 装订规范
- 分类清晰
- 编号统一
- 便于检索

## 12.5 信息系统文档国家标准

### 一、核心讲解

#### 1. GB/T 8567-2006

**软件文档管理指南**，规定了14种文档：

| 文档名称 | 说明 |
|----------|------|
| 可行性研究报告 | 项目立项依据 |
| 项目开发计划 | 项目总体计划 |
| 需求规格说明书 | 系统功能和性能要求 |
| 数据要求说明书 | 数据需求 |
| 概要设计说明书 | 系统总体设计 |
| 详细设计说明书 | 模块详细设计 |
| 数据库设计说明书 | 数据库设计 |
| 用户手册 | 用户使用指南 |
| 操作手册 | 操作人员指南 |
| 测试计划 | 测试策略和方法 |
| 测试分析报告 | 测试结果分析 |
| 开发进度月报 | 月度进展报告 |
| 项目开发总结报告 | 项目总结 |
| 软件维护手册 | 维护指南 |
