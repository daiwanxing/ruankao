## 一、教学目标

掌握事务处理系统（TPS）、管理信息系统（MIS）、决策支持系统（DSS）以及各种专用信息系统的详细特点和组成。

## 二、核心讲解

### 1. 事务处理系统（TPS）

**定义**：
对企业管理中日常事务所发生的数据进行输入、处理和输出。

**TPS数据处理周期五个阶段**：

| 处理阶段 | 内容 |
|---------|------|
| **数据输入** | 第一阶段，数据输入方式有三种：人工、自动、二者结合 |
| **数据处理** | 批处理（定期处理，无法实时）；联机事务处理OLTP（立即处理，成本高） |
| **数据库的维护** | 对数据库的访问形式：检索、修改、存入、删除 |
| **文件报表的产生** | TPS的输出就是为终端用户提供所需的有关文件和报表 |
| **查询处理** | 支持批次查询或联机实时查询，通过屏幕显示获得查询结果，对不同级别用户授予不同访问权限 |

**开发方法**：
TPS面对的是结构化程度很高的管理问题，因此可以采用结构化生命周期法来进行开发。

**企业资源规划（ERP）功能**：
1. 支持决策的功能
2. 为处于不同行业的企业提供有针对性的IT解决方案
3. 从企业内部的供应链发展为全行业和跨行业的供应链

### 2. 管理信息系统与决策支持系统

**管理信息系统（MIS）**：

**系统分类**：
- **闭环系统**：计算机实时处理的系统
- **开环系统**：批处理系统

**决策支持系统（DSS）**：

**基本结构形式**：
1. **两库结构**
2. **基于知识的结构**

**DSS的组成**：
1. 数据的重组和确认
2. 数据字典的建立
3. 数据挖掘和智能体
4. 模型建立

### 3. 专用信息系统

**(1) 知识管理系统（KSM）**

**知识管理关键**：获取、存储、共享和使用知识

**知识存储**：知识被创建后，通常存储在包含文档、报告、文件和数据库的知识库中

**(2) 专家系统（ES）**

**特点**：
1. 求解的问题是半结构化或非结构化问题
2. 是人类专家在问题领域的推理，而不是模拟问题领域本身
3. 包含数据级、知识库级和控制级三级知识
4. 面对的往往是实际的问题，而不是纯学术的问题
5. 问题求解的通用性较差

**组成**：
1. 知识库
2. 综合数据库
3. 推理机
4. 知识获取
5. 解释程序
6. 人机接口

**(3) 虚拟现实系统（VRS）**

**定义**：
使一个或多个用户能够在计算机模拟环境中移动和反应。虚拟现实模拟需要特殊的接口设备，将模拟世界的景象、声音和感觉传送给用户。

**(4) 办公自动化（OA）系统**

**主要功能**：
1. 事务处理
2. 信息管理
3. 辅助决策

**组成**：
1. 计算机设备
2. 办公设备
3. 数据通信及网络设备
4. 软件系统

## 三、通俗例子

<Card title="TPS例子" icon="information">
就像银行的存取款系统，客户来存取钱（数据输入），系统立即处理并更新账户余额（数据处理），维护客户数据库（数据库维护），打印凭条（文件报表产生），客户可以查询余额（查询处理）。
</Card>

<Card title="OLTP例子" icon="information">
就像超市收银系统，每刷一件商品就立即计算总价并扣减库存，这就是联机事务处理，实时响应但系统负载大。
</Card>

<Card title="批处理例子" icon="information">
就像银行的夜间批量处理系统，把一天的交易记录汇总后统一处理，计算利息、更新账户，虽然不能实时但效率高。
</Card>

<Card title="DSS例子" icon="information">
就像公司的BI（商业智能）系统，收集销售数据、市场数据，经过分析建模后，为高层制定下季度销售策略提供决策支持。
</Card>

<Card title="ES例子" icon="information">
就像法律咨询机器人，用户咨询法律问题，系统根据法律知识库进行分析推理，给出专业的法律建议。
</Card>

<Card title="OA系统例子" icon="information">
就像钉钉的企业版，请假流程：员工提交申请→主管审批→系统自动通知→记录存档，这就是事务处理；同时还能查看公司通讯录（信息管理）；统计部门请假率（辅助决策）。
</Card>

## 六、重点总结

### 本节必记考点 ⭐⭐⭐⭐

1. **TPS五阶段**：数据输入→数据处理→数据库维护→文件报表→查询处理
2. **数据处理两方式**：批处理、OLTP（联机事务处理）
3. **DSS两结构**：两库结构、基于知识的结构
4. **ES五特点**：非结构化问题、专家推理、三级知识、实际问题、通用性差
5. **ES六组成**：知识库、综合数据库、推理机、知识获取、解释程序、人机接口
6. **OA三功能**：事务处理、信息管理、辅助决策
7. **MIS分类**：闭环系统（实时）、开环系统（批处理）
